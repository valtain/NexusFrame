git config user.name 으로 현재 사용자명을 확인해줘.

git log 를 사용하여 오늘 날짜 기준 커밋 목록을 hash, 시간, 메시지와 함께 표시해줘.
기본값은 현재 사용자의 커밋만 표시 (--author 필터 적용).
커밋 목록 하단에 "[전체 보기]" 선택지도 함께 제공해줘.

날짜 범위(예: 2일치) 또는 commit hash 직접 입력도 받아줘.

선택이 완료되면 다음 순서로 진행해:

1. 선택한 커밋들의 diff 를 읽어 변경 내용과 설계 의도를 파악해
   - commit message 뿐 아니라 실제 코드 변경 내용을 기반으로 해석할 것

2. 아래 규칙으로 업무 보고서를 작성해:

[포맷 규칙]
- 최상위 고정 제목: * 3D RPG 게임 Framework 작업 중
- 하위 항목은 한 단계씩 들여쓰기, 아래 구조를 따를 것:
    * 섹션명
      * 항목 — 설명
          - 세부 내용
- 코드 블록 안에 출력 (마크다운 렌더링 없이 일반 텍스트로 복사 가능하게)
- 날짜 표기 없음
- 세부 구현 목록 나열이 아닌 설계 의도와 구조적 결정 중심으로 서술
- 짧고 간결하게, 명사형 종결

[섹션 분류 기준]
- 새 기능/시스템 구현 → 기능명 그대로 섹션 사용 (예: GamePlaySystem 구현)
- 검증/테스트 환경 → "검증 환경 구성"
- 도메인 용어 정립, API 명칭 정비 → "설계 개선" (설계 개선과 코딩 컨벤션은 반드시 별도 섹션으로 분리)
- 네이밍, 포맷 등 컨벤션 정비 → "코딩 컨벤션 적용"
